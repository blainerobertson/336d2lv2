The Accounts Folder

You have previously created a controller for the acme site itself, but to provide a more focused approach to managing visitor accounts (e.g. logging in and registering), we will create a folder and new controller specifically for that purpose.

Create the Accounts folder

  1. Within the acme folder, create a new folder named "accounts". Do you remember the web naming rules?
    • No capital letters
    • No spaces in folder or file names
    • No special characters in the name
    • Names should be short and meaningful
  2. Now that we have the folder we need a controller for this folder!
  3. To make things easier (we hope) for ourselves, we will copy, paste and edit to do this.

Create the Controller

Because the acme controller (index.php) at the root of the acme site has the basics we will use it to create our new accounts controller.

  1. Copy the acme controller (index.php) at the root of the acme site.
  2. Click the accounts folder and paste.
  3. An index.php file should now appear inside of accounts.

Edit the Controller

Alter the Requires

  1. Open the accounts controller.
  2. Change the comment at the top of the page to reflect that this is the "Accounts Controller".
  3. We have to edit the paths to the database connection and acme model files.
  4. Look at the acme folder structure. You will see that the model and library folders are siblings to the accounts folder.
  5. Since our file is inside the accounts folder, we have to tell the server to move up one level, then find them. To do this:
    • Add "../" in front of both paths so that they look like the examples below:
      require_once '../library/connections.php'; require_once '../model/acme-model.php';
    • Keep in mind that your file names may be different than what is shown here.

Keep the navList Code

Below the two require_once statements is the code you wrote earlier to build the navigation list.

Keep it as you will probably be needing it again.

Alter the Action Collection Code

You should have a sequence of code to collect and store the value from the action name - value pair. It should look like the code example below:

$action = filter_input(INPUT_POST, 'action');
if ($action == NULL) {
 $action = filter_input(INPUT_GET, 'action');
 if ($action == NULL) {
  $action = 'home';
 }
}

We are going to leave most of it as it, but alter the last assignment statement. In this case, if there is no specific action we will provide a value to deliver the acme login view. (Remember in the MVC we always send requests to the controller and let it deliver the view). Alter the code so that it now looks like the code below:

$action = filter_input(INPUT_POST, 'action');
if ($action == NULL) {
 $action = filter_input(INPUT_GET, 'action');
 if ($action == NULL) {
  action = 'signin');
  }
}

Alter the Control Structure

Finally we get to the switch control structure. For right now, leave the first case statement, but delete all the rest. When you're done it should look like this:

switch ($action) {
  case 'home':
   include 'view/home.php';
   break;
}

What's Next?

In Enhancement 3 you will create links pointing to this new "accounts" controller with name - value pairs that instruct the controller to deliver the login view and registration view. The code that currently exists in this controller will be modified and added to in order for the views to be delivered.
