Importing a SQL File

As mentioned in the "Create a Local Database" file once a database is created the next step is to create tables and fields, then populate the tables with data. While this can be done by hand, a much faster approach if available, is to import SQL files that have the table creation queries prewritten. When imported, these script files will be read by the database server and the queries used to create the tables and data if part of the SQL file.

The reverse is also possible. A SQL file can be exported from an existing database and then imported to a new database. Thus, saving time in the transferring of the database tables and data from one source to another.

The directions below guide you through the process of importing a SQL file to create two tables and populate the tables with data for the acme database (which should have been created in the "Creating a Local Database" activity.

Download the SQL File

  1. Download the zip file containing the chapter and exercise files for the text.
  2. Unzip the downloaded file.

Launch the XAMPP Servers

  1. Start XAMPP.
  2. Start the Apache Web server
  3. Start the MySQL server

Open The PHPMyAdmin Tool

  1. With the Apache and MySQL servers running, open a browser window or ne tab.
  2. Type "http://localhost/phpmyadmin" into the browser address bar.

Import the SQL File

  1. At the server level (the default when the PHPMyAdmin tool opens), click the "Import" tab.
  2. Click the "Choose File" button.
  3. Navigate to your Downloads folder and to the unzipped murach_files folder.
  4. Open the book_apps folder.
  5. Open the _create_db folder.
  6. Click the create_db.sql file. Click "Open".
  7. Scroll to the bottom of the Import screen and click the "Go" button.
  8. If things work correctly, a green bar should appear indicating that the import was successful and 23 queries were executed.

Check the Databases

  1. On the left, you should see a new item named "My".
  2. Click the "plus" to expand it.
  3. Beneath it you should see a new item named "guitar".
  4. Click the "plus" to expand it.
  5. You should now see two new databases - my_guitar_shop1 and my_guitar_shop2.
    6. You can now click either of the databases to view the tables and data within the database.

As mentioned earlier, these databases accompany the example applications found in the Murach textbook. You can use these databases to work along with the chapter examples.

One Change

In the chapters, the authors may refer to an administrator table and fake users within that table. The table was omitted from this import. Use the proxy user you created in the "Create a Local Database" activity instead when working with these databases.
