Accounts Model

This activity will have you create a new model and function, specifically for handling database interactions concerning registering site visitors to the acme web site. Later it will have additional functionality added to it.

Remember that in an Model - View - Control architecture the model is specifically for storing functions that allow for interacting with the database. In previous work you created a model for the acme site and built a function to select category data in order to build a dynamic navigation bar.

Create the Model

  1. Find your model folder in the acme web site.
  2. Right-click on the folder and select "New" > "PHP file".
  3. Name the file "accounts-model.php" so that it's clearly distinguised from any and all other models that exist or may be created later.

Add the Insert Function

  1. Open the accounts-model.php page.
  2. Edit the default comment to indicate that this is the accounts model for site visitors.
  3. Add several blank lines then add another comment to indicate that the new function will handle site registrations
  4. Create a new function whose name is regVisitor().
  5. You will recall that in our registration view we ask for four pieces of information:
    • First name
    • Last name
    • Email address
    • Password
  6. Each of these pieces of information will be stored into a variable and these variables will have to be supplied to this function as arguments (sometimes called parameters).
  7. Create four variables (representing the data just discussed) and place them inside the parentheses, separated by commas and spaces.
  8. When done, your code should resemble the illustration below: 
    function regVisitor($firstname, $lastname, $email, $password){

  }

Build the Prepared Statement

When you built the previous function to select the categories data you used a prepared statement. We will do so again. In this case, the prepared statement will be doubly important because of its added security feature to protect our database from SQL Injection attacks. This, because we are accepting data sent from site visitors and we don't know if they are honest folks or hackers!

The simple rule of web security is, "Never Trust Incoming Data!

The function code is below:

function regVisitor($firstname, $lastname, $email, $password){
// Create a connection object using the acme connection function
   $db = acmeConnect();
// The SQL statement
   $sql = 'INSERT INTO clients (clientFirstname, clientLastname,
           clientEmail, clientPassword)
           VALUES (:firstname, :lastname, :email, :password)';
// Create the prepared statement using the acme connection
   $stmt = $db->prepare($sql);
// The next four lines replace the placeholders in the SQL
// statement with the actual values in the variables
// and tells the database the type of data it is
   $stmt->bindValue(':firstname', $firstname, PDO::PARAM_STR);
   $stmt->bindValue(':lastname', $lastname, PDO::PARAM_STR);
   $stmt->bindValue(':email', $email, PDO::PARAM_STR);
   $stmt->bindValue(':password', $password, PDO::PARAM_STR);
// Insert the data
   $stmt->execute();
// Ask how many rows changed as a result of our insert
   $rowsChanged = $stmt->rowCount();
// Close the database interaction
   $stmt->closeCursor();
// Return the indication of success (rows changed)
   return $rowsChanged;
}

An Explanation

  • $db = acmeConnect(); - creates a new database connection object by calling the acme database connection function from the connections.php file.
  • $sql = 'INSERT INTO clients (clientFirstname, clientLastname, clientEmail, clientPassword) VALUES (:firstname, :lastname, :email, :password)'; - this is the SQL INSERT statement, but using named parameters instead of actual values as part of the prepared statement.
  • $stmt = $db->prepare($sql); - sends the SQL statement to the database server where it is checked for correctness, and if it is, a PDO Prepared Statement object is created and stored into the $stmt variable.
  • $stmt->bindValue(':firstname', $firstname, PDO::PARAM_STR); - each of these four bindValue functions replaces the name parameter (e.g. :firstname) with the actual value from the variable (e.g. $firstname). It also tells the database the type of data it is receiving (e.g. PDO::PARAM_STR), in this case a string.
  • $stmt->execute(); - This sends the now complete SQL statement to the server and the data is inserted to the database.
  • $rowsChanged = $stmt->rowCount(); - The prepared statement object asks the database server to indicate how many rows changed as a result of the last SQL query, the number that is returned is stored into the $rowsChanged variable. We anticipate that number being 1, meaning 1 new record was added to the database.
  • $stmt->closeCursor(); - Closes the interaction between the function and the database server.
  • return $rowsChanged; - Sends the value of the $rowsChanged variable back to wherever the function was called (this should be in the controller).

Save the File

The function is finished. Be sure to compare your code (particularly your acme connection function name) against the function name used in this sample code. Be sure to save the file.
