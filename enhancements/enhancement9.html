Enhancement 9

Overview

In this enhancement you are asked to implement enhanced functionality dealing with image uploads:

  1. Change the target of image uploads to the "products" folder within the acme "images" folder from the current "uploads > images" folder.
  2. Upload all existing images in the "images > products" using the upload process so that the images will be sized correctly and stored into the "images" table of the acme database.
  3. Write code that will find and deliver any product images from the database to the products "detail" view when a product is selected via the main menu of the Acme site.

Video Overview

This short video demonstrates what the finished enhancement will look like and the functionality it should possess.

Tasks

  1. During the activities for this week all image uploads are directed to the "images" folder inside the "uploads" folder. This was great for testing, but not so good for the Acme site. Therefore:
    1. Change the target path to point to the "products" folder within the main "images" folder at the root of the acme site.
    2. The change will occur in the "uploads" controller.
    3. Once made, run the file upload tool to make sure your new images are being stored to the correct location.
  2. The current image files in the "images > products" folder are not sized correctly, nor do they appear in the "images" table of the acme database. We want both to occur, therefore:
    1. Copy the existing "images > products" folder and store it on your desktop.
    2. Delete all existing images in the original "images > products" folder.
    3. Use the image upload application to upload the original images and associate them with the correct product. The thumbnails do not need to be uploaded as new thumbnails will be created as part of the upload process.
    4. When done, use the main navigation bar to click each category item and make sure the thumbnail images of each product appear in the view. Note: The inventory table thumbnail image path for the anvil items, refers to ".gif" images, these two will need to be changed to ".png" images.
    5. Then click, each product link and make sure the "full-size" image loads in the product detail view.
    6. This can be checked by using the "inspect" tools in the browser to make sure the "full size" images are no larger than 500 pixels wide or high and that thumbnail images are no larger than 200 pixels wide or high.
  3. When a "product detail" view is loaded the "full size" image should display, but what if additional images have been uploaded for that product? It would be beneficial if "thumbnails" of all images pertaining to that item were displayed. Therefore:
    1. In the uploads model, build a new function that will obtain thumbnail image information from the "images" table that are based on the productId. You will need to use the LIKE operator in your SQL query to find "-tn" in the file name.
    2. In the products controller, call the function when the product detail page is requested. The uploads model will have to brought into the controller's scope.
    3. The returned list of thumbnail images should be fed into a custom function that will wrap them in HTML so they are ready to be viewed. No additional information about the image needs to be included, just the thumbnail image itself with an "alt" value added to the image.
    4. The HTML containing the images should be included into the "product detail" view by the "products" controller when the view is delivered and styled to be professional in appearance.
    5. Add two images (it doesn't matter if they actually match the product) to at least two inventory items (products).
    6. To test, access the "product detail" view via the Acme navigation bar. Make sure the additional images appear and are professional in their appearance. The view should continue to be valid HTML5 and valid CSS3 when done.
