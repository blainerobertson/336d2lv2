Enhancement 3

Purpose

In this enhancement you will combine skills from the last two weeks to build a system into the acme web site that is capable of delivering both a registration and login view to a site visitor when the "My Account" link in the page header on the Acme site is clicked. This process will require use of the MVC architecture.

Tasks

  1. Work with your learning team to accomplish the tasks and post to and read the weekly discussion board to help one another.
  2. Use your acme template to build a login view. The form will require the user to supply:
    • their email address
    • their password

    Make sure you use HTML5 and usability standards to build the view and form (e.g. use label elements with the input elements and use HTML5 input types for the input elements within the form).

    Use CSS to style the form for professional appearance and usability.

    The form DOES NOT NEED TO ACTUALLY LOGIN AN INDIVIDUAL for this enhancement.

  3. You will use your acme template to build a registration view. The view must contain a form that collects the following information from a site visitor:
    • First name
    • Last name
    • Email address
    • A password

    Ensure that the registrant understands that all fields are required.

    Make sure you use HTML5 and usability standards to build the view and form (e.g. use label elements with the input elements and use HTML5 input types for the input elements within the form).

    Use CSS to style the form for professional appearance and usability.

    The form DOES NOT NEED TO ACTUALLY REGISTER AN INDIVIDUAL for this enhancement.

  4. Create a link in the acme template and the existing acme views (home, login, 500 error and registration) for the "My Account" item in the page header. The link must:
    • point to the acme controller, and
    • send a key - value pair (remember the "action" key from the activities?) as a parameter that tells the controller to deliver the login view.

    You may want to carefully study the links built in the dynamic navigation activity to see how the key-value pair is made part of the link.

  5. Create a link in the login view that tells the user to click it if they have not yet registered. The link must:
    • point to the acme controller, and
    • pass a key - value pair that tells the controller to deliver the registration view.

    Make sure the link is large and obvious so that it is not overlooked and causes a site visitor confusion. See this example from target.com:

Submission

  • Thoroughly test your links (in all views) and the revised header function (in the database connections file) to ensure that they point to the acme controller and that the correct views are delivered.
  • Validate all views for HTML5 validity.
  • Validate the CSS for validity.
  • Double check your work to ensure that it meets all of the expectations listed.
  • When satisfied that the code is operational:
    1. Create a video that demonstrates:
      1. That each of the links in the views work.
      2. That the home, login and registration views are valid html and css.
    2. Post your video to your YouTube channel and copy the link to the video when published.
    3. Zip the acme folder and name it enhancement 3 (save this zip file outside of your web site)
    4. Submit the zip file to the enhancement 3 assignment dropbox and add the video URL to the dropbox comment.

Grading Matrix

This enhancement has points in Objectives 1 and 3.

  • All views are valid HTML5 - 5 points (minus 1 for each error found)
  • All views are valid CSS - 5 points (minus 1 for each error found)
  • The "My Account" link points to the controller and passes an appropriate key - value pair to request the login view and the view is delivered - 5 points (all or nothing)
  • The registration link in the login view points to the controller and passes an appropriate key - value pair to request the registration view and the view is delivered - 5 points (all or nothing)
  • The header functions in the database connection file point to the controller and passes an appropriate key - value pair to request the error view and the view is delivered - 5 points (all or nothing)
  • The acme controller has additional and operable case statements to deliver the appropriate views according to the value of the $action variable - 5 points (all or nothing)
  • Total value of objective 1: 10 points
  • Total value of objective 3: 20 points
