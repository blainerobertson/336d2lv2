Enhancement 3

Purpose

In this enhancement you will combine skills from the last two weeks to build a system into the acme web site that is capable of delivering both a registration and login view to a site visitor when the "My Account" link in the page header on the Acme site is clicked. This process will require use of the MVC architecture.

Video Demo

This short video demonstrates what your links and the flow of this enhancement could look like.

Tasks

  1. Work with your learning team to accomplish the tasks and post to and read the weekly discussion board to help one another.
  2. Make sure your acme template is using the "$navList" variable to provide the navigation bar as directed in the "Build a Dynamic Navigation Menu" activity, prior to doing step 3.
  3. Use your acme template to build a login view. The form will require the user to supply:
    • their email address
    • their password

    Make sure you use HTML5 and usability standards to build the view and form (e.g. use label elements with the input elements and use HTML5 input types for the input elements within the form).

    Use CSS to style the form for professional appearance and usability.

    The form DOES NOT NEED TO ACTUALLY LOGIN AN INDIVIDUAL for this enhancement.

  4. You will use your acme template to build a registration view. The view must contain a form that collects the following information from a site visitor:
    • First name
    • Last name
    • Email address
    • A password

    Ensure that the registrant understands that all fields are required.

    Make sure you use HTML5 and usability standards to build the view and form (e.g. use label elements with the input elements and use HTML5 input types for the input elements within the form).

    Use CSS to style the form for professional appearance and usability.

    The form DOES NOT NEED TO ACTUALLY REGISTER AN INDIVIDUAL for this enhancement.

  5. Create a link in the header code snippet module for the "My Account" item. By doing it here, the link should appear in the acme template and all existing views (home, login, 500 error and registration).
  6. The link must point to the "accounts" controller and send a name-value (a.k.a. key - value) pair (remember the "action" name or key from the activities?) as a parameter that tells the controller to deliver the login view. You may want to carefully study the links built in the dynamic navigation activity to see how the name-value pair is made part of the link.
  7. Alter the accounts controller to make the delivery request of the login view the default value of the $action variable (if no other value is found).
  8. Alter the existing switch case statement in the accounts controller to react to the "login" request and deliver the login view from the view folder.
  9. Create a link in the login view that tells the client to click it if they have not yet registered or need to create an account. The link must:
    • point to the "accounts" controller, and
    • pass a name - value pair that tells the controller to deliver the registration view.

    Make sure the link is large and obvious so that it is not overlooked and causes a site visitor confusion. See this example from target.com:

  10. Add an additional case statement to the "accounts" controller to detect the request for the registration view and deliver it to the browser from the view folder.
  11. Save all of your work.

Test, Test, Test

  1. Make sure XAMPP is running.
  2. Run the acme main controller.
  3. Click the "My Accounts" link in the header of the home page. The login view should be delivered to the browser.
  4. Click the "Create a New Account" link. The registration view should be delivered to the browser.
  5. Make sure that the URL in both links is pointing to the accounts controller and passing name - value pairs as parameters to deliver the respective views. Under no circumstances should the links point directly to the view!
  6. Test the "My Account" link in all views to make sure it works in all of them.
  7. If things work "Clap your hands and shout Hooray!".
  8. If things don't work, go back and troubleshoot your code, ask for help, post to the discussion board, but GET HELP until it works.

Submission

  • Thoroughly test your links to ensure that they point to the accounts controller and that the correct views are delivered.
  • Validate all views for HTML5 validity.
  • Validate the CSS for validity.
  • Double check your work to ensure that it meets all of the expectations listed.
  • When satisfied that the code is operational:
    1. Create a video that demonstrates:
      1. That each of the links in the views work.
      2. That the home, login and registration views are valid html and css.
    2. Post your video to your YouTube channel and copy the link to the video when published.
    3. Zip the acme folder and name it enhancement 3 (save this zip file outside of your web site)
    4. Submit the zip file to the enhancement 3 assignment dropbox and add the video URL to the dropbox comment.

Grading Matrix

Objective 1 Items

  • All views are valid HTML5 - 5 points (minus 1 for each error found)
  • All views are valid CSS - 5 points (minus 1 for each error found)
  • Total value of objective 1: 10 points

Objective 3 Items

  • The "My Account" link points to the accounts controller and passes an appropriate name - value pair to request the login view and the view is delivered - 10 points (minus 3 points for each view that is missing this link)
  • The registration link in the login view points to the accounts controller and passes an appropriate name - value pair to request the registration view and the view is delivered - 5 points (all or nothing)
  • The accounts controller has additional and operable case statements to deliver the appropriate views according to the value of the $action variable - 5 points (all or nothing)
  • Total value of objective 3: 20 points
