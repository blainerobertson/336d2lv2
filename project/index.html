Dynamic Project

Description

The dynamic project is the major project of the semester and is inclusive of all topics and objectives of the course. The project must be built to the specifications provide to you. The following basics will be included:

  • Use of an M-V-C architecture (frameworks created by others are not allowed).
  • Views must be use semantic markup, responsive to large through small device screens and be valid HTML5 and CSS3;
  • Use custom functions to facilitate common code processes;
  • Use all four CRUD data processes involving the database;
  • Incorporate server and client-side data validation to protect the database and insure correct data submission;
  • Use the PDO library and prepared statements for database interactions within the model;
  • Build and use a database to match provided Entity Relation diagrams (ERDs). Unfamiliar with ERDs? Refer to Ch. 16 or https://www.youtube.com/watch?v=-fQ-bRllhXc).

Timeline

  1. Project docuement will be available at the beginning of week 12.
  2. Project will be due the last published day of classes for the semester - consult the course calendar.

Submission

  1. The code (in zip format) will be submitted to the project assignment dropbox on the last published day of classes by 12pm (Noon) Rexburg time.
    1. The zip file must be of the project folder containing all files needed for operation.
    2. A SQL file of the database (including tables and data) must be included in the zip file. The SQL file must be importable to a database of the name specified in the project documents and be capable of operation without any alteration.
  2. A video demonstrating your use of your project application must be created and the URL to the video submitted to the assignment dropbox (as a comment in the submission). The video is limited to 7 minutes.
